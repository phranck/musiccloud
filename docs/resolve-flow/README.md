# Resolve Flow

Architecture and data flow through the musiccloud resolve layer.

## PDFs

- **Deutsch**: [`de/resolve-flow.pdf`](de/resolve-flow.pdf)
- **English**: [`en/resolve-flow.pdf`](en/resolve-flow.pdf)

Each language has its own LaTeX source (`de/resolve-flow.tex`,
`en/resolve-flow.tex`) and its own D2 diagram sources
(`de/diagrams/`, `en/diagrams/`). The SVG renderings under each
`diagrams/` directory are committed alongside the sources so GitHub
can render them inline anywhere they are referenced.

The version stamp at the top of each PDF combines a manually
maintained semantic version (`VERSION` in this directory) with the
short Git commit hash and commit date of the most recent change to
`docs/resolve-flow/`.
